## 一、指標定義

| 指標 | 說明 | 用途 |
|------|------|------|
| **Close** | 原始 K 線收盤價 | 進場過濾 |
| **WMA_fast** | Close 的加權移動平均，預設週期 = 4 | 進場交叉 |
| **WMA_slow** | Close 的加權移動平均，預設週期 = 6 | 進場交叉 |
| **swing high** | 原始 K 線「前 N 根」最高價（含訊號 bar，N=4） | Stage 1 初始止損 |
| **Bollinger Band** | 中軌 = WMA(close, 20)，上下軌 = ±2σ；σ 以中軌為基準 | Stage 3 拖曳止損（r_ladder 模式下不使用） |

**注意**：ATR 已**不再使用於止損計算**（v2.0 舊版 Chandelier 已被 v2.1 三階段取代）。

## 二、建倉條件（Entry）

必須**同時滿足**以下兩個條件，才在下一根 K 線開盤時掛限價空單：

### 條件 1 — 趨勢過濾（死亡交叉）

- **Bar[0]** 收盤後：`WMA_fast[0] < WMA_slow[0]`
- **Bar[-1]**（前一根）：`WMA_fast[-1] >= WMA_slow[-1]`
- 即 WMA_fast 由上往下穿越 WMA_slow，形成**死亡交叉**

### 條件 2 — 趨勢結構確認

- `Close[-2] > Close[0]`
- `Close[-3] > Close[0]`
- 即交叉前第 2、第 3 根的平均K收盤價皆高於交叉當根，確認趨勢在交叉前已持續下跌

### 建倉動作

- 在 **Bar[1]** 開盤時，以**限價單**開空單
- 限價價格 = 真實K線 Bar[1] 開盤價 × (1 − 滑點 0.03%)
- 滑點方向往下，確保在開盤瞬間能快速成交（marketable limit）
- 若成交後 swing-based 初始止損仍 ≤ 成交價（極端窄幅 K 線情況），broker 會放棄該筆進場

```
時間軸示意：

... | Bar[-3] | Bar[-2] | Bar[-1] | Bar[0] 死叉發生（收盤後判斷）| Bar[1] 進場 |
       ↑           ↑                        ↑                          ↑
  Close[-3]  Close[-2]           條件1判斷                  限價賣出
  需 > Close[0]  需 > Close[0]                        = 真實開盤 × 0.9997
```

## 三、三階段止損策略（Stop Loss）

止損採用**狀態機式三階段設計**，每階段 stop 永遠只能往有利方向移動（ratchet：空單只能下移）。

### Stage 1 — 初始保護（剛進場）

```
stop = max(high over [t-3 .. t]) × (1 + stage1_slippage_buffer)
     = 訊號 bar 含當下「前 4 根」原始 K 線最高點，再往上 0.03% buffer
```

- 用 swing high 提供寬鬆的初始保護，避免一進場就被噪音打掉
- 0.03% 滑點 buffer 推離極值，降低誤觸機率
- 此 stop 在 Stage 2 觸發前不會更新

### Stage 2 — 鎖利保本

**觸發條件**：價格朝有利方向（往下）走到 `stage2_trigger_r × R`

定義：
- `R = |entry_price − Stage 1 initial_stop|`（風險距離）
- `progress = (entry_price − bar.low) / R`（空單）

| R 模式 | 條件 | 觸發點 |
|--------|------|--------|
| Normal | `R ≥ 2 × taker + slippage`（成本距離） | **1.2R** |
| Abnormal | `R < 2 × taker + slippage` | **2.4R**（×2） |

**新止損計算**：
```
stop = entry_price × (1 − 2 × taker_fee_rate − slippage_pct) − buffer_r × R
     = 進場價 − 雙向交易成本 − 0.2R 額外 buffer
```

設計動機：到 1.2R 後拉保本，但純粹「保本+成本」會在價格小幅回拉時立即平倉、吃不到趨勢；
加上 0.2R buffer 降低被掃機率。

### Stage 3 — 趨勢跟蹤（Bollinger）

**觸發條件**：價格朝有利方向走到 `stage3_trigger_r × R`

| R 模式 | 觸發點 |
|--------|--------|
| Normal | **2.4R**（=Stage 2 normal trigger × 2）|
| Abnormal | **4.8R**（=Stage 2 abnormal trigger × 2）|

**新止損計算**：
```
candidate = min(Stage 2 fixed stop,  Bollinger upper band)
```

- Bollinger upper = `WMA(close, 20) + 2σ`（用**原始** K 線）
- Stage 2 fixed 永遠當作 ceiling：即使 Bollinger upper 暫時高於保本 stop，也不會把保護升到保本之上
- Bollinger upper 隨價格下跌緩緩下降，跌破 ceiling 後接管 → 自然 ratchet

### Ratchet 規則（所有階段共通）

```python
if candidate < current_stop:    # 空單只往下動
    update_stop(candidate)
# else: 維持原值，不退讓
```

- Bollinger 回勾上拉 → stop 不動
- 價格回升 → stop 不動
- 只有當新計算值「更有利（更低）」才更新

## 四、止盈策略（Take Profit）

**沒有固定止盈點**，全部由三階段 stop 帶出：

- Stage 1：被掃 → 小虧（接近 −R）
- Stage 2：被掃 → 接近保本（含成本與 0.2R 緩衝）
- Stage 3：被掃 → 鎖到一定獲利後出場

讓獲利自然奔跑，由 Bollinger 的上軌決定何時停利。

## 六、邏輯流程圖

```mermaid
flowchart TD
    A[開始監控] --> B{Bar[0] 收盤<br/>WMA_fast 下穿 WMA_slow？}
    B -- 否 --> A
    B -- 是 --> C{Close[-2] > Close[0]<br/>且 Close[-3] > Close[0]？}
    C -- 否 --> A
    C -- 是 --> D[Bar[1] 開盤<br/>限價空單<br/>= 開盤 × 0.9997]
    D --> E[Stage 1<br/>stop = max high [t-3..t] × 1.0003<br/>R = |entry − stop|]
    E --> F{R 模式判斷}
    F -- R ≥ 成本 --> F1[normal: <br/>stage2 trigger=1.2R<br/>stage3 trigger=2.4R]
    F -- R < 成本 --> F2[abnormal: <br/>stage2 trigger=2.4R<br/>stage3 trigger=4.8R]
    F1 --> G{每根 K 收盤}
    F2 --> G
    G --> H{progress ≥ stage2_trigger?}
    H -- 否 --> J{價格觸及 stop？}
    H -- 是 --> I[進入 Stage 2<br/>stop = entry × 0.9987 − 0.2R]
    I --> K{progress ≥ stage3_trigger?}
    K -- 否 --> J
    K -- 是 --> L[進入 Stage 3<br/>stop = min stage2_fixed, BB_upper]
    L --> M{Bollinger 下移？}
    M -- 是且 candidate < current --> N[ratchet stop ↓]
    M -- 否或回勾 --> J
    N --> J
    J -- 否 --> G
    J -- 是 --> O[市價平倉出場]
    O --> A
```

## 七、與舊版 v2.0 的差異

| 項目 | v2.0（已棄用） | v2.1（目前） |
|------|---------------|--------------|
| 止損型態 | 單一 Chandelier Exit | 三階段狀態機 |
| 公式 | `lowest(N) + ATR×k` | Stage 1 swing → Stage 2 保本 → Stage 3 BB |
| 主要參數 | `atr_period`, `atr_multiplier`, `atr_lookback` | swing/Stage 2/Stage 3 全套 |
| 鎖利機制 | 無（純跟蹤）| Stage 2 顯式保本 + Stage 3 BB 跟蹤 |
| Cost-aware | 否 | abnormal_R 模式自動加倍 trigger |
